You can inspect and modify your model manually in two ways:
1. Inspect your model's outputs with a quick script
Run this standalone script to see what your model actually exposes:
Python
Copy
import openvino as ov
from pathlib import Path

model_path = "path/to/your/model.xml"

core = ov.Core()
model = core.read_model(str(model_path))
compiled = core.compile_model(model, "CPU")

print("=== INPUTS ===")
for inp in compiled.inputs:
    print(f"  Name: {inp.get_any_name()}")
    print(f"  Shape: {inp.shape}")
    print(f"  Type: {inp.element_type}")
    
print("\n=== OUTPUTS ===")
for i, out in enumerate(compiled.outputs):
    print(f"  [{i}] Name: {out.get_any_name()}")
    print(f"      Shape: {out.shape}")
    print(f"      Type: {out.element_type}")

Quick checklist for debugging a new model
Table
Symptom	Likely cause	Fix
Crash at compiled.output("pred_score")	Wrong output name	Use index compiled.output(0) or add your name to the keyword list
Heatmap is solid color	Map not normalized or wrong shape	Add debug prints, check squeeze() behavior
Score is always 0 or huge	Wrong output picked as score	Verify with inspection script which output is scalar
Image looks stretched/distorted	Wrong input size	Hardcode H, W to training size
Heatmap misaligned with image	Resize interpolation	Change cv2.resize interpolation method
